Dos and Don'ts for Entering Equations

Do's Don'ts
Use mathematical operators: +, -, *, / Use unsupported operators like ^ for exponentiation.
Use exp(t) for et. Use e^t for exponential.
Use sin(t), cos(t), tan(t) for trigonometric functions. Use sint, cost, tant, or misspell function names.
Use log(t) for natural logarithm (ln). Use log10(t) for base-10 logarithm (use log(t, 10) instead).
Use simple variable names like t, x, y, n. Use spaces in variable names, e.g., x y or t n.
Use parentheses to group expressions correctly. Forget parentheses when needed, such as exp(t + 1) instead of exp(t + 1.
Use ** for exponentiation, e.g., t**2 for t2. Use ^ for exponentiation, e.g., t^2.
Input expressions as valid Python-style syntax (e.g., t**2 + 2*t + 1). Use unsupported syntax or functions (e.g., x(t) + sin(t without proper closure).
Input functions like sin(t) or cos(t) directly. Use unsupported forms like sin(t + 1) in an incorrect format.
Use a simple structure like x(t) + exp(t). Enter complex expressions without proper function definitions or parentheses.
